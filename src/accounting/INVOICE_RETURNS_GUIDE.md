# دليل نظام إرجاع الفواتير

# Invoice Returns System Guide

## نظرة عامة / Overview

نظام إرجاع الفواتير يسمح بإرجاع كامل أو جزئي للأصناف من فواتير البيع أو الشراء مع إنشاء قيود محاسبية عكسية تلقائية.

The invoice returns system allows full or partial returns of items from sales or purchase invoices with automatic reverse accounting entries.

---

## المزايا / Features

✅ **إرجاع جزئي أو كامل** - Return full or partial quantities
✅ **قيود محاسبية عكسية تلقائية** - Automatic reverse accounting entries
✅ **تحديث المخزون** - Automatic inventory updates
✅ **تتبع الإرجاعات** - Return history tracking
✅ **حماية من الأخطاء** - Validation and error protection

---

## كيفية العمل / How It Works

### 1. فاتورة البيع الأصلية / Original Sales Invoice

عند إنشاء فاتورة بيع بقيمة **500 دينار**:

```
القيد المحاسبي الأصلي:
┌─────────────────────────┬────────┬────────┐
│ الحساب                  │ مدين   │ دائن   │
├─────────────────────────┼────────┼────────┤
│ الصندوق/الذمم المدينة   │ 500    │ -      │
│ إيرادات المبيعات        │ -      │ 500    │
└─────────────────────────┴────────┴────────┘
```

### 2. إرجاع جزئي بقيمة 200 دينار / Partial Return of 200

عند إرجاع أصناف بقيمة **200 دينار**:

```
القيد المحاسبي العكسي:
┌─────────────────────────┬────────┬────────┐
│ الحساب                  │ مدين   │ دائن   │
├─────────────────────────┼────────┼────────┤
│ إيرادات المبيعات        │ 200    │ -      │
│ الصندوق/الذمم المدينة   │ -      │ 200    │
└─────────────────────────┴────────┴────────┘
```

**النتيجة النهائية:**

- الصندوق: 500 - 200 = **300 دينار**
- إيرادات المبيعات: 500 - 200 = **300 دينار**

## استخدام الـ API / API Usage

### إرجاع فاتورة بيع / Return Sales Invoice

**Endpoint:**

```
POST /accounting/sales-invoices/:id/return
```

**Request Body:**

```json
{
  "salesInvoiceId": 1,
  "items": [
    {
      "originalSalesInvoiceItemId": 5,
      "productId": 10,
      "quantity": 2,
      "unitPrice": 100,
      "unitId": 1,
      "toWarehouseId": 1,
      "itemNote": "منتج معيب"
    }
  ],
  "note": "إرجاع بسبب عيب في المنتج",
  "returnReason": "عيب في التصنيع",
  "referenceNumber": "RET-S20250930-001"
}
```

**Response:**

```json
{
  "returnInvoice": {
    "id": 15,
    "invoiceNumber": "RET-S20250930-001",
    "totalAmount": -200,
    "status": "PAID"
  },
  "originalInvoice": {
    "id": 1,
    "invoiceNumber": "S20250930-001",
    "totalAmount": 500,
    "remainingAmount": 300
  },
  "totalReturnAmount": 200,
  "message": "تم إرجاع الأصناف بنجاح"
}
```

## القيود المحاسبية حسب النوع / Accounting Entries by Type

### 1. فاتورة بيع نقدية / Cash Sales Invoice

**الفاتورة الأصلية:**

- **مدين:** الصندوق
- **دائن:** إيرادات المبيعات

**عند الإرجاع:**

- **مدين:** إيرادات المبيعات (تقليل الإيرادات)
- **دائن:** الصندوق (إرجاع المبلغ للعميل)

---

### 2. فاتورة بيع آجلة / Credit Sales Invoice

**الفاتورة الأصلية:**

- **مدين:** الذمم المدينة (حساب العميل)
- **دائن:** إيرادات المبيعات

**عند الإرجاع:**

- **مدين:** إيرادات المبيعات (تقليل الإيرادات)
- **دائن:** الذمم المدينة (تقليل دين العميل)

---

### 3. فاتورة شراء نقدية / Cash Purchase Invoice

**الفاتورة الأصلية:**

- **مدين:** المخزون/المشتريات
- **دائن:** الصندوق

**عند الإرجاع:**

- **مدين:** الصندوق (استرداد المبلغ)
- **دائن:** المخزون/المشتريات (تقليل المخزون)

---

### 4. فاتورة شراء آجلة / Credit Purchase Invoice

**الفاتورة الأصلية:**

- **مدين:** المخزون/المشتريات
- **دائن:** الذمم الدائنة (حساب المورد)

**عند الإرجاع:**

- **مدين:** الذمم الدائنة (تقليل دين المورد)
- **دائن:** المخزون/المشتريات (تقليل المخزون)

---

## تحديث المخزون / Inventory Updates

### إرجاع فاتورة بيع:

- ✅ **إضافة** الكمية المرتجعة للمخزن المحدد
- ✅ زيادة رصيد المنتج في المخزن

### إرجاع فاتورة شراء:

- ✅ **طرح** الكمية المرتجعة من المخزن المحدد
- ✅ التحقق من توفر الكمية قبل الإرجاع

---

## التحقق من الصحة / Validation

النظام يتحقق من:

1. ✅ وجود الفاتورة الأصلية
2. ✅ وجود الأصناف في الفاتورة الأصلية
3. ✅ الكمية المرتجعة لا تتجاوز الكمية الأصلية
4. ✅ توفر الكمية في المخزن (للشراء)
5. ✅ صحة المخزن المحدد

---

## أمثلة عملية / Practical Examples

### مثال 1: إرجاع جزئي لفاتورة بيع

**السيناريو:**

- فاتورة بيع بقيمة 1000 دينار
- 5 منتجات × 200 دينار
- إرجاع منتجين فقط (400 دينار)

**النتيجة:**

- فاتورة إرجاع بقيمة -400 دينار
- الفاتورة الأصلية المتبقي: 600 دينار
- المخزن: +2 وحدة

---

### مثال 2: إرجاع كامل لفاتورة شراء

**السيناريو:**

- فاتورة شراء بقيمة 3000 دينار
- 10 منتجات × 300 دينار
- إرجاع كامل (10 منتجات)

**النتيجة:**

- فاتورة إرجاع بقيمة -3000 دينار
- الفاتورة الأصلية المتبقي: 0 دينار
- المخزن: -10 وحدات
- استرداد كامل المبلغ من المورد

## ملاحظات مهمة / Important Notes

⚠️ **فواتير الإرجاع:**

- تُنشأ كفواتير منفصلة بمبالغ سالبة
- تُربط بالفاتورة الأصلية
- حالتها تلقائياً "مدفوعة" (PAID)

⚠️ **القيود المحاسبية:**

- تُنشأ تلقائياً عند الإرجاع
- تعكس القيود الأصلية بالضبط
- تستخدم نفس الحسابات من Invoice Config

⚠️ **المخزون:**

- يتم تحديثه فوراً عند الإرجاع
- يجب التأكد من توفر الكمية (للشراء)
- يمكن إرجاع البضاعة لمخزن مختلف
