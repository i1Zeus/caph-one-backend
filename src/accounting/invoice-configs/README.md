# نظام تكوين الحسابات المحاسبية للفواتير

## نظرة عامة

يوفر هذا النظام إمكانية تكوين الحسابات المحاسبية التلقائية لكل نوع من أنواع الفواتير (مبيعات/مشتريات) ونوع الدفع (نقدي/آجل).

## كيفية العمل

### 1. إعداد التكوين

يتم إنشاء تكوين لكل مجموعة من:
- **نوع الفاتورة**: مبيعات (SALES) أو مشتريات (PURCHASE)
- **نوع الدفع**: نقدي (CASH) أو آجل (CREDIT)
- **الحساب المدين**: الحساب الذي سيتم خصمه
- **الحساب الدائن**: الحساب الذي سيتم إضافته إليه

### 2. أمثلة على التكوينات

#### فاتورة مشتريات نقدي:
- **المدين**: حساب المشتريات (EXPENSE)
- **الدائن**: حساب الصندوق (ASSET - Cash)
- **المعنى**: زيادة المشتريات، نقص النقد

#### فاتورة مشتريات آجل:
- **المدين**: حساب المشتريات (EXPENSE)
- **الدائن**: حساب الموردين (LIABILITY)
- **المعنى**: زيادة المشتريات، زيادة الالتزامات

#### فاتورة مبيعات نقدي:
- **المدين**: حساب الصندوق (ASSET - Cash)
- **الدائن**: حساب المبيعات (REVENUE)
- **المعنى**: زيادة النقد، زيادة الإيرادات

#### فاتورة مبيعات آجل:
- **المدين**: حساب العملاء (ASSET - Receivables)
- **الدائن**: حساب المبيعات (REVENUE)
- **المعنى**: زيادة المستحقات، زيادة الإيرادات

## الاستخدام في الكود

### إنشاء تكوين جديد

```typescript
const config = await invoiceConfigsService.create({
  invoiceType: InvoiceType.PURCHASE,
  paymentType: PaymentType.CASH,
  debitAccountId: 1,  // حساب المشتريات
  creditAccountId: 2, // حساب الصندوق
  description: 'فاتورة شراء نقدي',
  isActive: true
});
```

### إنشاء معاملة تلقائية

```typescript
await invoiceConfigsService.createAutomaticTransaction(
  InvoiceType.PURCHASE,
  PaymentType.CASH,
  1000, // المبلغ
  'فاتورة شراء #123',
  supplierId,
  invoiceId
);
```

## قاعدة البيانات

### جدول `invoice_accounting_configs`

| العمود | النوع | الوصف |
|--------|-------|--------|
| id | INT | المعرف الفريد |
| invoiceType | ENUM | نوع الفاتورة (SALES/PURCHASE) |
| paymentType | ENUM | نوع الدفع (CASH/CREDIT) |
| debitAccountId | INT | معرف الحساب المدين |
| creditAccountId | INT | معرف الحساب الدائن |
| description | STRING | وصف التكوين |
| isActive | BOOLEAN | فعال أم لا |

### قيود قاعدة البيانات

- **Unique Constraint**: (`invoiceType`, `paymentType`) - تكوين واحد لكل نوع فاتورة ونوع دفع
- **Foreign Keys**: `debitAccountId` و `creditAccountId` مرتبطان بجدول `accounts`

## API Endpoints

### GET `/accounting/invoice-configs`
استرجاع جميع التكوينات

### POST `/accounting/invoice-configs`
إنشاء تكوين جديد

### GET `/accounting/invoice-configs/:id`
استرجاع تكوين محدد

### PATCH `/accounting/invoice-configs/:id`
تحديث تكوين موجود

### DELETE `/accounting/invoice-configs/:id`
حذف تكوين

## الأمان والتحقق

1. **التحقق من وجود الحسابات**: يتم التأكد من وجود الحسابات المدين والدائن
2. **منع التكرار**: لا يمكن إنشاء تكوين مكرر لنفس نوع الفاتورة ونوع الدفع
3. **التحقق من صحة البيانات**: التأكد من صحة أنواع البيانات المدخلة
4. **المعاملات الآمنة**: استخدام Database Transactions لضمان سلامة البيانات

## المعالجة الاحتياطية

إذا لم يتم العثور على تكوين مناسب، سيتم استخدام المنطق الاحتياطي القديم لضمان عدم توقف النظام.
