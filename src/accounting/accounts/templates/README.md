# نظام القوالب المحاسبية (Business Templates)

## نظرة عامة

يوفر نظام القوالب المحاسبية قوالب جاهزة ومُحددة مسبقاً لشجرة الحسابات المحاسبية حسب نوع العمل. هذا يساعد المستخدمين على البدء بسرعة مع حسابات محاسبية مناسبة لنوع أعمالهم دون الحاجة لإنشاء الحسابات يدوياً.

## الميزات الرئيسية

- ✅ 7 قوالب جاهزة لأنواع أعمال مختلفة
- ✅ حسابات محاسبية مُحددة مسبقاً حسب نوع العمل
- ✅ تكوينات تلقائية للفواتير
- ✅ واجهة مستخدم تفاعلية لاختيار القالب
- ✅ إمكانية تخصيص بادئة الحسابات
- ✅ التحقق التلقائي من وجود حسابات مسبقة

## القوالب المتاحة

### 🏪 محل تجاري (Retail Store)
**الحسابات المضمنة (19 حساب):**
- أصول: صندوق، بنك، عملاء، مخزون، ديكور
- خصوم: موردون، إيجار مستحق، رواتب
- حقوق ملكية: رأس المال، أرباح محتجزة
- إيرادات: مبيعات، خصومات
- مصروفات: تكلفة بضاعة، رواتب، إيجار، كهرباء، إعلان

### ☕ مقهى (Cafe)
**الحسابات المضمنة (20 حساب):**
- أصول متخصصة: آلات قهوة، طاولات وكراسي، ديكور
- مخزون مُقسم: قهوة ومواد، حلويات
- إيرادات متنوعة: مشروبات، طعام، حلويات
- مصروفات خاصة: موسيقى وترفيه، تنظيف

### 🛒 متجر إلكتروني (E-commerce)
**الحسابات المضمنة (18 حساب):**
- أصول رقمية: بوابات دفع إلكتروني، برامج المتجر
- إيرادات: مبيعات إلكترونية، مردودات
- مصروفات متخصصة: استضافة، رسوم بوابات دفع، إعلانات رقمية

### 🍔 مطعم (Restaurant)
**الحسابات المضمنة (17 حساب):**
- مخزون مُقسم: مواد غذائية، مشروبات
- موظفين متخصصين: طهاة، خدمة
- إيرادات: طعام، مشروبات، رسوم خدمة

### 🏥 عيادة طبية (Medical Clinic)
**الحسابات المضمنة (17 حساب):**
- أصول طبية: أجهزة طبية، مخزون أدوية
- إيرادات طبية: كشف طبي، أدوية، تحاليل
- موظفين: أطباء، تمريض
- مصروفات متخصصة: تأمين طبي، صيانة أجهزة

### 🚗 ورشة سيارات (Auto Repair Shop)
**الحسابات المضمنة (16 حساب):**
- مخزون متخصص: قطع غيار، زيوت وسوائل
- أدوات ومعدات: معدات ورشة، أدوات إصلاح
- خدمات: إصلاح، قطع غيار، صيانة

### 💇 صالون تجميل (Beauty Salon)
**الحسابات المضمنة (16 حساب):**
- مخزون تجميل: مستحضرات، أدوات تجميل
- خدمات متنوعة: قص شعر، تجميل، عناية بالبشرة
- معدات متخصصة: معدات صالون، ديكور وإضاءة

## كيفية الاستخدام

### 1. من خلال الواجهة (Frontend)
- عند عدم وجود حسابات محاسبية، سيظهر تلقائياً منتقي القوالب
- اختر القالب المناسب لنوع عملك
- أدخل بادئة مخصصة (اختياري)
- اضغط "تطبيق القالب"

### 2. من خلال API

#### عرض جميع القوالب
```http
GET /accounting/accounts/templates
```

#### عرض قالب محدد
```http
GET /accounting/accounts/templates/retail-store
```

#### تطبيق قالب
```http
POST /accounting/accounts/templates/apply
Content-Type: application/json

{
  "templateId": "cafe",
  "customPrefix": "C1"
}
```

## إضافة قالب جديد

لإضافة قالب جديد، قم بتعديل ملف `business-templates.ts`:

```typescript
const newTemplate: BusinessTemplate = {
  id: 'pharmacy',
  name: 'صيدلية',
  nameEn: 'Pharmacy',
  description: 'قالب للصيدليات ومحلات الأدوية',
  icon: '💊',
  accounts: [
    // قائمة الحسابات...
  ],
  invoiceConfigs: {
    salesCash: { debitCode: '1100', creditCode: '4100' },
    // باقي التكوينات...
  }
};

// إضافة القالب للقائمة
export const businessTemplates: BusinessTemplate[] = [
  // القوالب الموجودة...
  newTemplate,
];
```

## القواعد والقيود

1. **لا يمكن تطبيق القالب إلا عند عدم وجود حسابات محاسبية**
2. **يتم إنشاء عملة رئيسية تلقائياً (الدينار العراقي) إذا لم توجد**
3. **جميع الحسابات تُربط بالعملة الرئيسية افتراضياً**
4. **تكوينات الفواتير التلقائية تُنشأ تلقائياً مع القالب**

## استكشاف الأخطاء

### خطأ: "يوجد حسابات مسبقاً"
**السبب**: يمكن تطبيق القالب فقط على نظام فارغ
**الحل**: احذف جميع الحسابات أولاً أو ابدأ نظام جديد

### خطأ: "لم يتم العثور على القالب"
**السبب**: معرف القالب غير صحيح
**الحل**: تحقق من قائمة القوالب المتاحة

### خطأ في إنشاء تكوينات الفواتير
**السبب**: الحسابات المطلوبة للتكوين غير موجودة
**الحل**: تحقق من صحة أكواد الحسابات في القالب

## API Reference

| Endpoint | Method | الوصف |
|----------|--------|--------|
| `/accounting/accounts/templates` | GET | جلب جميع القوالب |
| `/accounting/accounts/templates/:id` | GET | جلب قالب محدد |
| `/accounting/accounts/templates/apply` | POST | تطبيق قالب |
| `/accounting/accounts/check-empty` | GET | التحقق من وجود حسابات |

## الأمان

- جميع العمليات مُسجلة في نظام التدقيق
- التحقق من صحة البيانات قبل الإنشاء
